# 本地端口规划

本文档记录 AeroForgeMES 在本地开发环境中的监听端口。除前端开发服务器外，前端请求统一经由网关转发。

## 应用服务

| 服务 | 模块 | 本地地址 | 端口 | 说明 |
| --- | --- | --- | --- | --- |
| 前端开发服务器 | `frontend` | `http://localhost:3000` | 3000 | Vite 开发服务器；`/api` 默认代理到网关。 |
| API 网关 | `mes-gateway` | `http://localhost:8080` | 8080 | 前端 API 的统一入口，转发至后端业务服务。 |
| 认证服务 | `mes-auth` | `http://localhost:8081` | 8081 | 登录、令牌与认证相关接口。 |
| 系统服务 | `mes-system` | `http://localhost:8082` | 8082 | 用户、角色、菜单等系统管理接口。 |
| 设备服务 | `mes-equipment` | `http://localhost:8083` | 8083 | 设备台账、维护与监控接口。 |
| 报表服务 | `mes-report` | `http://localhost:8086` | 8086 | 报表与指标相关接口。 |
| 集成服务 | `mes-integration` | `http://localhost:8087` | 8087 | ERP、OpenAPI 与数据同步接口。 |
| 生产服务 | `mes-production` | `http://localhost:9103` | 9103 | 生产订单、任务、安灯与条码相关接口。 |
| 质量服务 | `mes-quality` | `http://localhost:9104` | 9104 | 质量计划、检验与缺陷管理接口。 |

## 基础设施

| 服务 | 本地地址 | 端口 | 说明 |
| --- | --- | --- | --- |
| MySQL | `localhost:3306` | 3306 | 各后端模块默认连接本机 MySQL；数据库按服务划分为 `mes_auth`、`mes_system`、`mes_equipment`、`mes_report`、`mes_integration`、`mes_production` 和 `mes_quality`。 |

## 调用关系

```text
浏览器 → 前端（3000） → 网关（8080） → 各业务服务
```

前端也可通过环境变量 `VITE_GATEWAY_URL` 覆盖默认网关地址；默认值为 `http://localhost:8080`。

## 配置来源

- 前端端口与代理：`frontend/vite.config.ts`
- 网关及各业务服务端口：各模块的 `backend/*/src/main/resources/application.yml`
